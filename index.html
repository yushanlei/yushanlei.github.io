Apple Watch behavior design

01

Ablaze 着了

  • Apple Watch
  • SwiftUI
  • Fire System
  • Social UX
  • Vibe Coding

一个让拖延有体感代价的 Apple Watch 应用：当你忽视洗手、deadline 或正念任务，手腕上会燃起一团珊瑚色的火。

Ablaze 的核心是人格化惩罚：火知道你忽视过几次，文案会从“该洗手了。”逐步收紧到“洗手”。

MVP 验证统一火焰视觉、今日账本，以及 deadline / 洗手 / 正念的阶段机制；久坐火因与 Apple Stand 重合，被收进实验功能。

社交版本只显示朋友“正在被什么烧”的抽象状态。互相纵火、送粉色火、下浅灰雾，都是对提醒权力的产品化处理。

论文 DDL
点击灭火，完成这团 DDL

它把提醒从一条通知变成一个会变化的状态：拖延会升温，完成会灭火，朋友也可以介入。

Cultural AI guide

02

Lu Xun AI Tour Guide 鲁迅 AI 导览

  • Multimodal AI
  • React
  • Anthropic API
  • Cultural Heritage

一个微信式 AI 导览界面：拍下鲁迅相关历史地点，得到带有文学脉络与时代语境的对话式解释。

微信式 UI 是产品判断：让用户在熟悉的交互里进入陌生的历史。

多模态输入之后，Claude 不只识别图像，也被提示去保留历史准确性与对话温度。

关键不在回答更多问题，而在把历史地点从“可打卡的纪念物”重新组织成一段可追问的现场。

鲁迅 AI 导览
这栋楼和鲁迅有什么关系？
这里可以从“生活现场”而不只是“文学纪念地”来理解。鲁迅的文本常常把街巷、房间、饭桌这些普通空间变成时代情绪的入口。

进入视口后，对话会按真实聊天节奏浮现。

Personal memory product

03

Personalized Memory Engine 个性化 Memory 系统

  • AI Product Design
  • User Profiling
  • Prototype
  • Xiaohongshu

为小红书 AI 聊天机器人“点点”设计的记忆系统：从搜索、点赞、收藏、评论、关注与购买里，生成可被对话调用的用户画像。

难点是“记住到什么程度才让人觉得被理解，而不是被盯着”。

她设计信号权重、画像结构和对话唤起方式，再用 vibe coding 做出可试用原型。

这个项目连接了用户心理、平台伦理与产品落地：从人文判断到工程交付之前的那一步。

为了验证“记忆如何被体验”，她又做了两个轻量活动原型。

认识你自己 know-yourself

这个活动把用户画像翻译成四种可感知结果：颜色、味道、动物与风景；再延伸到生活哲学、点点眼中的你，以及一个会从旧收藏里翻出线索的信箱。

  • 你的四重奏
  • 你的生活哲学
  • 点点眼中的你
  • 你的信箱
Prototype result
等待生成 点击下方按钮，生成一个用户画像示例；“打开信箱”会用点击模拟摇一摇，从尘封收藏里找东西。
我们之间 between-us

这个活动把 memory 从“平台如何认识我”推进到“我和另一个人如何彼此理解”：默契匹配给出七个维度、歌曲与一句话，懂我测试用十道题检验对 TA 的认识。

  • 默契匹配
  • 你懂我吗
  • 7 个关系维度
  • 10 道个性化题
Prototype result
等待匹配 点击生成一个两人关系示例，展示画像如何被转译成互动结果。
对话输出
“我记得你最近常看城市漫步，也收藏过几家老书店。要不要把今天的路线排得慢一点？”

数据流不是监控的隐喻，而是一套关于分寸感的产品设计。
